# The Story Nook

A picture-book creation and reading app. The admin portal lets you upload page photos and generate a complete illustrated book using Claude AI. The public library lets readers browse and read the finished books.

## Architecture

```
picture-books/
├── frontend/   # React + Vite — public library & reader
└── backend/    # Express + HTMX + Handlebars — admin portal & API
```

| Concern | Technology |
|---|---|
| Public reader | React 18, TypeScript, Vite, Tailwind CSS v4 |
| Admin portal | Express, Handlebars, HTMX 2, Tailwind CSS v4 |
| AI generation | Google Gemini (`gemini-2.0-flash`) |
| Storage | JSON file (`backend/data/books.json`) + local uploads |

## Prerequisites

- [nvm](https://github.com/nvm-sh/nvm) with Node 26 (`nvm install 26`)
- A Google AI Studio API key ([get one here](https://aistudio.google.com/apikey))

## Setup

```bash
# Use the right Node version
nvm use 26

# Install all dependencies
npm run install:all

# Configure environment
cp backend/.env.example backend/.env
# → Edit backend/.env and set ANTHROPIC_API_KEY
```

## Development

```bash
# Start both frontend (port 5173) and backend (port 3000) together
./dev.sh

# Or individually:
cd backend && npm run dev    # Express + Tailwind watch on :3000
cd frontend && npm run dev   # Vite dev server on :5173
```

The frontend dev server proxies `/api` and `/uploads` to the backend, so everything works from `http://localhost:5173`.

The admin portal is at `http://localhost:3000/admin`.

## Production build

```bash
npm run build   # Compiles frontend to backend/public (served by Express)
npm start       # Runs the Express server
```

## How it works

1. Open the **Admin Portal** (`/admin`) and fill in a title, age range, and illustration style.
2. Optionally upload one photo per page spread.
3. Click **Analyze & Generate** — the backend sends the content to Claude, which writes the story text and generates illustration descriptions. Progress is shown via HTMX polling.
4. Once complete, preview each spread, edit text inline, request new images, reorder or split spreads.
5. Click **Publish to Library** to make the book visible in the public reader.

## Project structure

```

├── server.js               # Express app, Handlebars setup
├── routes/
│   ├── admin.js            # HTMX routes, job store, Claude orchestration
│   └── api.js              # Public REST API (/api/books)
├── services/
│   ├── bookStore.js        # JSON file CRUD
│   └── claude.js           # Anthropic SDK wrapper
├── views/
│   ├── layouts/admin.hbs   # HTML shell, HTMX, global JS
│   ├── admin.hbs           # Main admin page
│   └── partials/           # HTMX-swapped fragments
├── public/
│   ├── css/admin.css       # Generated by Tailwind CLI (git-ignored)
│   └── uploads/            # Uploaded & generated images
├── data/books.json         # Book storage
└── tailwind.input.css      # Tailwind source for admin CSS

frontend/
├── src/
│   ├── main.tsx            # Entry point
│   ├── App.tsx             # Root component (library ↔ reader routing)
│   ├── types.ts            # Shared TypeScript interfaces
│   └── components/
│       ├── Library.tsx     # Book grid, fetches /api/books
│       ├── BookCard.tsx    # Single book tile
│       ├── Reader.tsx      # Full-screen spread reader
│       └── CornerFrame.tsx # Decorative corner overlay
└── vite.config.ts
```
